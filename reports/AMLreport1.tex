When obtaining data, it is not uncommon for some parts of it to go missing. An interesting case is when it happens specifically to the labels. In this project, we focus on such scenarios. We take into account four distinct options, when the labels missing are:

\begin{enumerate}
    \item fully random,
    \item dependent on a feature,
    \item dependent on all features,
    \item dependent on all features and the label itself.
\end{enumerate}

It is important to evaluate how to handle such cases, detect dependence (if such exists), and fill the missing data back in. 

In this project we inspect the performance of two label inferring methods and compare them with the naive method (removing all unlabeled observations) and the oracle method (an optimistic case in which no labels are missing) in all four missing label scenarios. Then, we fit our own implementation of logistic regression with Lasso (using FISTA optimizer) and \verb|sklearn|'s \verb|LogisticRegression| with L1 penalty and compare the results.

\section{Datasets selection and cleanup}

After careful consideration we've selected the following four datasets:

\begin{enumerate}
    \item Shopping habits of people who shop in store vs. people who shop online,
    \item phone battery condition and suggested approach,
    \item software quality prediction based on its properties,
    \item wine quality data.
\end{enumerate}

In these four we predict where someone shops, whether a phone needs its battery changed, if a piece of code is defective, and if a specific wine is of good quality, respectively.

All four datasets underwent a cleanup process. Luckily none of them had any NaNs (or None cells) nor duplicated records. The target features were cast to binary (some of them were multivariate, others binary labels defined by strings). Finally some discrete string features were cast to integers. All this has been done with a few helper functions and \verb|pandas|'s \verb|info()| method.

\section{Optimization with FISTA}

% \textcolor{red}{Found the link to the paper which introduces FISTA: https://www.ceremade.dauphine.fr/~carlier/FISTA}

Fast Iterative Shrinkage-Thresholding Algorithm (FISTA in short) has been proposed in 2009 as a way to minimize functions of form $f = g + h$, where $g$ is a smooth convex function and $h$ is a continuous, but not necessarily smooth, convex function \cite{fista}. In the case of Lasso in Logistic Regression, the loss function has the form of

$$F(x) = \frac{1}{n} \sum_{i=1}^n \log(1 + e^{-y_i x_i^T \beta}) + \lambda\|\beta\|_1,$$

where $\lambda > 0$. It is a known fact that $\|\cdot\|_1$ is continuous, convex, and non-smooth, which fits into FISTA assumptions.

Our implementation of the FISTA algorithm for Lasso Regression is based on \cite{fista_algo} and combines the already known ISTA with an approach called acceleration to achieve quick convergence. There are a few key parts of our implementation which are worth discussing.

The soft-thresholding function \verb|ST| can be expressed using a following formula:

$$\textrm{ST}_l(\beta) = \textrm{sgn}(\beta) \max\{|\beta| - l, 0\}$$

Intuitively, it takes a vector $\beta$ and reduces the absolute value of each coordinate by a maximum of $l$. This is a special case of proximal mapping, which, more generally, allows to minimize $f$ easily. Additionally, we have defined two helper functions \verb|sigmoid| and \verb|gradient| corresponding to $\sigma$ and $\nabla \mathcal{L}$, respectively, where $\mathcal{L}$ is the logistic loss function.

Next, we perform gradient downscaling. We start with step size $t = L^{-1}$, where $L$ is the Lipschitz constant, approximated as $$L = \frac{\|X\|^2_2}{4n}.$$ 

In the $k$-th iteration, firstly, an accelerated parameter vector $v$ is calculated using the following formula:

$$v = \beta_{k-1} + \frac{k-1}{k+2}(\beta_{k-1} - \beta_{k-2}).$$

We use it to calculate the potential candidate for the new parameters. We correct the vector $v$ by subtracting the gradient of $\nabla\mathcal{L}$ at $v$ multiplied by the step size $t$. 

Subsequently, we calculate the next step candidate $\beta_k = \textrm{ST}_{\lambda t}(v_{corrected})$.

Finally, we check if the following condition is satisfied:

$$\|\beta_k - \beta_{k-1} \| < 1\mathrm{e}{-6} \cdot (1 + \|\beta_{k-1} \|)$$

If it does, we stop the optimization process; otherwise, we perform another step.

\section{Labeling missing data}

We have implemented two methods that focus on filling missing labels: Linear Discriminant Analysis and K Nearest Neighbors.

\subsection{LDA}

This approach is based on an assumption that data belonging to the same class should be close to each other. In other words, observations from each class should form a cluster. We assume that observations form two Gaussian clusters. Using labeled data, we estimate parameters of distributions: two mean vectors (one for each class) and the between group covariance matrix. Then we fill remaining labels by maximizing the posterior probability of a class given a set of features.

Since LDA generates a linear decision boundary, we could replace it with a standard linear model which we would use to predict missing labels. On one hand, this approach would not require to make assumptions of gaussian distributions of data. On the other hand, however, generating labels from a linear model to fit a different linear model might reinforce inaccuracies of the first one, especially if label missingness depends or features or labels themselves.

\subsection{KNN}

Instead of predicting distributions of data for each class, we can look more locally, at the neighborhood of the point with a missing label. The reasoning behind this method is quite simple -- data with similar features should intuitively belong to the same class.

We suspect that the quality of this solution may depend on the choice of the \verb|n_neighbors| parameter which, in turn, might be different for each dataset. Nevertheless, we decided to use the default value of 5 to simplify the solution.

\section{Results}

In this section, we show the results of our experiments and provide commentary.

\subsection{Comparison with sklearn}

Firstly, we compared our implementation of Lasso Logistic Regression with \verb|LogisticRegression| implemented in \verb|sklearn| by plotting profile plots and F1 plots for both classes. We selected a range of $\lambda$ parameters individually for each dataset. Then, we fit models for each $\lambda$ using unremoved labels. The results are shown in Figures \ref{fista_sklearn_shopping}, \ref{fista_sklearn_smartphone}, \ref{fista_sklearn_software} and \ref{fista_sklearn_wine} in Appendix \ref{figures_implementations}.

In profile plots, we can see that parameter trajectories are similar, although shifted, for both implementations. It is worth noting that our FISTA implementation shrinks parameters quicker (with respect to $\lambda$) than \verb|sklearn|'s. Additionally, there is some unexpected behavior in case of shopping and software datasets -- the parameter values seem to be shrunk for small lambdas. Nevertheless, these results are quite satisfying.

Plots of F1 scores mostly behave in an expected manner -- if we compare implementations, the trajectories of the measure for different splits are similar, although shifted -- just like in profile plots.

The notable exception is the software dataset, for which the F1 measure is constant and equal to 0 along all splits and implementations. As it turned out, both logistic regression models predicted all labels as 0. We inspected the split (especially the prior probabilities of each class in each sample), performed it with a different seed and looked at different lambda parameters and did not find a reason for this behavior. Despite this, we still decided to include it in the report, since the profile plot itself looks sensible and aids us in drawing conclusions.

\subsection{Missing labels}

Our next set of experiments focused on seeing how percentage of missing labels affects performance of our FISTA implementation and how it can be mitigated by label imputation methods.

We have decided to perform these experiments on the wine dataset, since F1 trajectories calculated for it showed the most interesting behavior. The results are shown in Figures \ref{wine_mcar}, \ref{wine_mar1}, \ref{wine_mar2}, and \ref{wine_mnar} in Appendix \ref{figures_missing_labels}. 

These plots are unexpected for us. First of all, we assumed that the Oracle approach would always yield the best results, since it has access to undisturbed labels. This is not the case: all metrics seem to have similar values and F1 trajectories fluctuate around each other without any noticeable pattern.

Second of all, we were sure that removing more labels would result in severe decrease of performance. Instead, F1 doesn't change significantly with the rise of missing label percentage. The MNAR scheme is a barely noticeable exception -- on the plot shown in Figure \ref{wine_mnar} we can see that for non-Oracle imputation methods, the metric decreases slightly.

Lastly, it seems that no matter the label missingness scheme, we achieved similar F1 scores as in the base model fitted for full data (as shown in Figure \ref{fista_sklearn_wine}). This provokes a question whether label imputation is that important if the dataset is not significantly small. 

\section{Conclusions}

In this project, we addressed binary classification with missing labels by combining three components: (i) data preparation and simulation of four missingness mechanisms (MCAR, MAR1, MAR2, MNAR), (ii) our own implementation of logistic Lasso regression optimized with FISTA, and (iii) a two-method label completion framework (LDA and KNN) wrapped in \verb|UnlabeledLogReg| and compared against Naive and Oracle baselines.

From the optimization perspective, our FISTA implementation produced coefficient paths and validation behavior qualitatively consistent with \verb|sklearn| logistic regression with L1 penalty, which supports the correctness of the implementation. The main observed difference was a shift in regularization strength at which shrinkage occurred, suggesting that while both approaches recover similar trends, practical model selection remains sensitive to scaling, stopping criteria, and parameterization details.

From the missing-label perspective, the expected strong dominance of Oracle over the remaining methods was not consistently observed in the wine experiments. Across MCAR, MAR1, and MAR2, all approaches often yielded similar F1 values, and only MNAR showed a clearer degradation trend for non-Oracle methods as the missingness level increased. This indicates that for this dataset and feature space, moderate label incompleteness can be partially compensated by the available covariate information and by simple completion rules.

Comparing the two completion strategies, no single method dominated in every setting. This confirms that the quality of label imputation is mechanism-dependent: local neighborhood assumptions (KNN) and distributional assumptions (LDA) may each be beneficial depending on the geometry of the data. Therefore, treating imputation as a tunable modeling choice, rather than a fixed preprocessing step, is important in practice.

An important limitation is that some observed trajectories (especially for the software dataset) were difficult to interpret, including degenerate predictions in both our and \verb|sklearn| implementations despite reasonable coefficient profiles. This suggests potential class imbalance or split-specific effects and motivates additional robustness checks (repeated splits, class-balanced resampling, and threshold analysis) in future work.

Overall, the project objectives were achieved: we implemented and validated FISTA-based logistic Lasso, proposed and tested two methods for handling missing labels, and performed comparative experiments under four missingness schemes and multiple missingness levels. The key practical conclusion is that the impact of missing labels is highly context dependent; in some regimes, simple completion methods can approach Oracle-level utility, while in MNAR scenarios careful modeling of the missingness mechanism becomes substantially more important.
